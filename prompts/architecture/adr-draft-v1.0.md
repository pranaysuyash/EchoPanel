# ADR Draft (EchoPanel) — v1.0

**Goal**: Draft an Architecture Decision Record for a meaningful technical/product decision.

---

## Inputs
- Decision title: `<short>`
- Decision context: `<why now>`
- Options considered: `<list>`
- Recommendation: `<one>`

---

## Output (required)
Create an ADR markdown with:
- Context
- Decision
- Options (pros/cons)
- Consequences (what we’re committing to)
- Follow-ups (tickets if needed)

Suggested location:
`docs/adr/ADR-<YYYYMMDD>-<slug>.md`

---

## Stop condition
Stop after the ADR draft (no implementation).
