# Feature PRD + Ticketing (EchoPanel) — v1.0

**Goal**: Turn a feature idea into a PRD and a set of implementable tickets for EchoPanel.

---

## Inputs
- Feature request: `<what the user wants>`
- Primary persona: `<who>`
- Surfaces: `macapp | server | landing`
- Constraints: `<privacy, offline, MVP>`

---

## Output (required)

### A) PRD
- Problem statement
- Target persona + core use cases
- Success metrics (qualitative + quantitative)
- User stories (5–10)
- UX flow (step-by-step) with states (empty/loading/error)
- Non-goals
- Risks (privacy, performance, trust)
- Rollout plan

### B) Ticket breakdown
Create 3–10 tickets in `docs/WORKLOG_TICKETS.md` with:
- Scope contract
- Acceptance criteria
- Verification steps (commands + manual)

---

## Stop condition
Stop after PRD + ticket set (no implementation).
