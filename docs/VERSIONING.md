# Versioning and Releases

## Versioning scheme
- Use SemVer: `MAJOR.MINOR.PATCH`
  - `MAJOR`: breaking protocol or storage changes
  - `MINOR`: new backwards-compatible features
  - `PATCH`: bug fixes

## v0.1 definition
- First usable end-to-end: capture, stream, render transcript, periodic cards/entities, export.

## Release checklist (minimal)
- Update `docs/SPEC_UPDATES.md`
- Verify notarization and code signing steps (when packaging is added)
- Verify permission onboarding copy is accurate for current macOS
