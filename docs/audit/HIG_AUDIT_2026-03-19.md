# macOS HIG Audit ‚Äî EchoPanel macapp_v2

**Audited:** `~/Projects/EchoPanel/macapp_v2/`
**Date:** 2026-03-19
**Standard:** Apple Human Interface Guidelines (macOS)

---

## Executive Summary

The v2 UI is architecturally solid and follows many HIG conventions, but has **16 distinct HIG violations** across 5 categories. Most are medium-severity. The most impactful: the window minimum size is too large for a panel app, the menu bar extra lacks a recent sessions list, onboarding can't be cancelled, and keyboard navigation is incomplete.

---

## 1. Window & Panel Design

### ‚úÖ Good
- `windowStyle(.titleBar)` ‚Äî correct for document-style app
- `defaultPosition(.center)` ‚Äî correct default placement
- `Settings` scene separated ‚Äî proper settings window pattern
- `NavigationView` with sidebar ‚Äî correct macOS layout pattern

### ‚öÝÔ∏è Issues

**Issue 1 ‚Äî Minimum window size too large** (Medium)
- **Location:** `EchoPanelV2App.swift:47`
- **Problem:** `frame(minWidth: 900, minHeight: 600)` ‚Äî this is a panel/menu-bar app. A 900√ó600 minimum is appropriate for a full document app, not a compact panel.
- **HIG ref:** "For a window whose content is scalable, set a reasonable minimum size ‚Äî often 400 by 300 points is sufficient."
- **Fix:** Reduce to `minWidth: 480, minHeight: 360` for compact panel use, or make it resizable without a minimum.

**Issue 2 ‚Äî Settings window too small** (Medium)
- **Location:** `SettingsView.swift`
- **Problem:** TabView frame is `500√ó380` ‚Äî settings windows should be at least 450pt tall. At 380pt + padding, essential settings content may be cut off.
- **HIG ref:** Settings windows should accommodate all content without scrolling if possible.
- **Fix:** Increase to `height: 450` minimum, or add content scrolling within tabs.

## 2. Menu Bar Extra

### ‚úÖ Good
- Standard menu bar icon (waveform SF Symbol)
- Version number shown
- Start/stop recording action visible
- Keyboard shortcut shown inline

### ‚öÝÔ∏è Issues

**Issue 3 ‚Äî No recording duration timer** (Medium)
- **Location:** `MenuBarView.swift`
- **Problem:** When recording, only a text label "You are recording" is shown. There's no elapsed time display.
- **HIG ref:** "Menu bar extras should convey essential state at a glance." Duration is critical state for a recording app.
- **Fix:** Add `Text(formatDuration(duration))` next to the status indicator, styled as monospaced.

**Issue 4 ‚Äî No recent sessions list** (Medium)
- **Location:** `MenuBarView.swift`
- **Problem:** Standard macOS menu bar extras (like Finder, Notes) show recent items below the main actions. EchoPanel shows only 3 actions (Start/End, Open Panel, Settings, Quit) ‚Äî no recent sessions.
- **HIG ref:** "Include recent items when appropriate so users can quickly access frequently used content."
- **Fix:** Add a "Recent Sessions" section below main actions showing last 3-5 sessions with titles and durations.

**Issue 5 ‚Äî No quit confirmation option** (Low)
- **Location:** `MenuBarView.swift`
- **Problem:** Quit is a single menu item with no confirmation if recording is active.
- **HIG ref:** "Warn users when quitting would interrupt an ongoing task."
- **Fix:** Show confirmation if `recordingState != .idle`.

## 3. Onboarding

### ‚úÖ Good
- 3-step flow is appropriately scoped
- Progress dots show current position
- "Get Started" CTA on final step
- Transitions are smooth (`withAnimation`)

### ‚öÝÔ∏è Issues

**Issue 6 ‚Äî No cancel/skip option** (Medium)
- **Location:** `OnboardingView.swift`
- **Problem:** No "Skip" or "X" button. Once started, user must complete or force-quit.
- **HIG ref:** "Let users skip setup steps they don't need." Users should be able to defer onboarding.
- **Fix:** Add `Button("Skip")` in the top-right corner, or make progress dots keyboard-navigable.

**Issue 7 ‚Äî Keyboard navigation incomplete** (Medium)
- **Location:** `OnboardingView.swift`
- **Problem:** Progress dots don't respond to arrow keys. Tab key doesn't cycle through steps.
- **HIG ref:** "All interactions should be accessible via keyboard."
- **Fix:** Add `.focusable()` to the VStack, handle `.onKeyPress(.leftArrow)` and `.onKeyPress(.rightArrow)` for step navigation.

**Issue 8 ‚Äî Accessibility labels missing on decorative elements** (Medium)
- **Location:** `OnboardingView.swift`, `ReadyStep`
- **Problem:** Circle icons (100pt decorative circles with SF Symbols) have no `accessibilityLabel`.
- **HIG ref:** VoiceOver must be able to announce the purpose of all interactive and meaningful elements.
- **Fix:** Add `.accessibilityLabel("Welcome illustration")` etc. to decorative ZStacks.

## 4. Navigation & Keyboard

### ‚úÖ Good
- Sidebar with `NavigationLink` ‚Äî correct macOS pattern
- `‚åò‚áßR/P/H` shortcuts implemented
- `.searchable(text:)` in sidebar ‚Äî correct placement
- `pickerStyle(.segmented)` for Highlights/Transcript/People tabs

### ‚öÝÔ∏è Issues

**Issue 9 ‚Äî LiveView tabs not keyboard-navigable** (Medium)
- **Location:** `LiveView.swift:32` (Picker)
- **Problem:** Picker uses `.pickerStyle(.segmented)` but the segment selection isn't reachable via arrow keys in a non-keyboard-focused context.
- **HIG ref:** "Support full keyboard navigation. Users should be able to reach every interactive element using Tab and arrow keys."
- **Fix:** Wrap the segmented control in a `Focusable` view, or ensure the Picker receives focus properly.

**Issue 10 ‚Äî No focus ring on custom buttons** (Low)
- **Location:** `MenuBarView.swift:44` (custom HStack buttons)
- **Problem:** `buttonStyle(.plain)` removes all styling including focus rings.
- **HIG ref:** macOS focus rings are essential for keyboard navigation.
- **Fix:** Add a custom `.buttonStyle` that shows a subtle focus ring when focused.

## 5. Visual Design & Accessibility

### ‚úÖ Good
- Uses `Material.thinMaterial` for toolbar ‚Äî appropriate glass effect
- `designTokens` with consistent spacing scale
- System semantic colors (`.accentColor`, `.red`)
- Monospaced font for timer display
- `.secondary` for supporting text

### ‚öÝÔ∏è Issues

**Issue 11 ‚Äî Custom color literals instead of system colors** (Low)
- **Location:** `DesignTokens.swift`, `StatusDot`
- **Problem:** Colors like `Color(hex: "22c55e")` for success/active should use `Color.green` or semantic system colors.
- **HIG ref:** "Use system colors that automatically adapt to Light/Dark mode."
- **Fix:** Replace `Color(hex: "22c55e")` with `.green`, `Color(hex: "ef4444")` with `.red`, etc.

**Issue 12 ‚Äî Empty state CTAs not prominent enough** (Low)
- **Location:** `EmptyStateView.swift`
- **Problem:** Empty states show a message but the primary CTA ("Get Started") isn't visually dominant.
- **HIG ref:** "Empty states should make clear what action to take."
- **Fix:** Use `.buttonStyle(.borderedProminent)` on the primary CTA, reduce secondary text prominence.

**Issue 13 ‚Äî Loading states don't use skeleton views consistently** (Medium)
- **Location:** `LoadingStateView.swift`, `LiveView`
- **Problem:** `SkeletonTranscriptCard` exists but `LiveView` shows a static "No highlights yet" during loading rather than skeleton cards.
- **HIG ref:** "Indicate progress for operations that take more than a moment."
- **Fix:** In `LiveView`, check `appState.isLoading` and show `SkeletonHighlightCard` views instead of static empty state.

## 6. Error Handling & Feedback

### ‚úÖ Good
- `ErrorStateView` with retry action
- Error type enum with specific cases
- `PausedStateView` for explicit paused state

### ‚öÝÔ∏è Issues

**Issue 14 ‚Äî Error messages lack specific recovery actions** (Medium)
- **Location:** `ErrorStateView.swift`
- **Problem:** Generic "ASR Error" / "LLM Error" ‚Äî doesn't tell user what to do. "Retry" button may not fix a credential issue.
- **HIG ref:** "Error messages should suggest how to resolve the problem."
- **Fix:** Add `resolution: String` to `AppError`, show context-specific recovery: "Check your API key in Settings ‚Üí Recording" for ASR errors.

**Issue 15 ‚Äî No toast/banner for transient successes** (Low)
- **Location:** `AppState.swift`
- **Problem:** When recording starts/stops, there's no visual confirmation beyond state change.
- **HIG ref:** "Provide feedback for user actions."
- **Fix:** Add a transient `.banner` or `Toast` for "Recording started", "Export complete", etc.

**Issue 16 ‚Äî Export has no progress indicator** (Low)
- **Location:** `ExportDialogs.swift`
- **Problem:** Export shows confirmation dialog but no progress view for large transcripts.
- **HIG ref:** "Show progress for operations taking more than a few seconds."
- **Fix:** Add `ProgressView("Exporting...")` with percentage for large exports.
