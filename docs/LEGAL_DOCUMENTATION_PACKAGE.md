## üîí **Privacy-First Architecture Documentation**

### **Core Privacy Principles**

#### **1. Local-First by Default**
- **Primary Processing:** All transcription happens on your Mac
- **Data Storage:** Audio and transcripts stored locally by default
- **No Cloud Dependency:** Core features work without internet
- **User Control:** Complete control over data retention and deletion

#### **2. Transparency in Design**
- **Clear Indicators:** Visual feedback for all data processing
- **Backend Status:** Always shows which transcription method is active
- **Cloud Activity:** Prominent indicators when cloud services are used
- **Recording State:** Clear indication when audio is being captured

#### **3. User Control and Choice**
- **Backend Selection:** Choose local-only or cloud processing
- **Feature Toggles:** Enable/disable OCR, speaker ID, AI analysis
- **Data Retention:** Configurable storage periods (0-365 days)
- **Export/Delete:** Full data portability and deletion rights

### **Security Documentation**

#### **Technical Security Measures**
- **Encryption:** Data encrypted at rest using macOS FileVault
- **Keychain:** Sensitive tokens stored in macOS Keychain
- **Sandboxing:** App limits data access to required resources only
- **Network Security:** HTTPS/TLS for all cloud communications

#### **Access Control**
- **macOS Permissions:** Screen recording and microphone governed by system
- **User Consent:** Explicit opt-in for all cloud services
- **Data Minimization:** Only collect data necessary for functionality
- **Local Processing:** Default transcription happens entirely on-device

## üåç **Regulatory Compliance**

### **GDPR Compliance (European Union)**

#### **Lawful Basis for Processing**
- **Consent:** Explicit opt-in to audio capture and processing
- **Legitimate Interests:** Providing core app functionality
- **Contract Necessity:** Fulfilling the service being used
- **User Rights:** Full access, rectification, and deletion rights

#### **GDPR User Rights**
1. **Right to Access:** Request copy of your stored data
2. **Right to Rectification:** Correct inaccurate personal data
3. **Right to Erasure:** Request complete data removal
4. **Right to Portability:** Export data for use in other services
5. **Right to Object:** Opt-out of certain data processing

### **CCPA Compliance (California)**

#### **California Consumer Rights**
- **Right to Know:** What personal data is collected and used
- **Right to Delete:** Request deletion of personal data
- **Right to Opt-Out:** Opt-out of data sales (we don't sell data)
- **Right to Non-Discrimination:** No discrimination for exercising rights

### **International Compliance**

#### **Multi-Jurisdictional Coverage**
- **United States:** Federal and state privacy laws
- **European Union:** GDPR compliance
- **Other Regions:** Compliance with local privacy regulations
- **Data Sovereignty:** User control over data location
